# MISREADER - 構想

## 一言で言うと

**投稿すると自動的にAIが3単語に要約してくれるSNS「MISREADER」**

**あなたの言葉を、AIが盛大に誤読するSNS**

---

## 解決したい課題

- SNSの投稿は長くなりがちで、流し読みでは要点をつかみにくい
- かといって短文制限があると、書きたい熱量や文脈が削られてしまう
- 「書く人は自由に長く書ける」「読む人は3単語でまず空気感をつかめる」を両立したい

## コアコンセプト

1. ユーザーは好きなだけ長文を書ける（最大5,000文字）
2. 投稿した瞬間、AIが意図的に詩的・哲学的などさまざまな方向で **誤読し、3単語に変換** する
3. タイムラインには **3単語の要約** が表示される
4. 元の文章は基本的に **投稿した本人だけ** が見られる
5. ただしその日の推薦数が最も多かった投稿は、翌日の `Daily Recommend` として原文が解放される

```
┌─────────────────────────────────┐
│ @username                       │
│                                 │
│ 鋭い / 皮肉 / わかる             │
│                                 │
│ [原文を見る]         2026/03/28 │
└─────────────────────────────────┘
```

## 特徴

- **書き手**: 最大5,000文字まで、十分な熱量で自由に書ける
- **読み手**: 3単語だけを手がかりに「何を言ったんだろう？」と想像する楽しさがある
- **AI**: 投稿のたびに自動で誤読し、詩的・哲学的・皮肉的に3単語へ変換する
- **推薦**: 各ユーザーは1日3回まで推薦を使え、1つの投稿に3回入れても、別々の投稿に分けてもよい
- **日次集計**: 推薦は日ごとに集計され、その日の最多推薦投稿が翌日に解放される
- **導線**: 翌日の当選投稿は `Daily Recommend` ページで確認できる
- **ブランド**: 読み違いさえ体験になる、少し挑発的な名前として `MISREADER` を採用

## 面白さのポイント

- **ズレが笑える**: 「今日眠い」→ 「魂の抵抗」みたいな、誤読の飛躍がまず面白い
- **想像が広がる**: 他人の3語から原文を勝手に補完したくなる
- **たまに怖いほど当たる**: AIの誤読が、意図より本質を突いてしまう瞬間がある
- **推し方に性格が出る**: 3回の推薦をひとつに集中させても、別々の投稿に配ってもいい
- **翌日が楽しみになる**: 「今日いちばん推された投稿が明日開く」という待ち時間が遊びになる

## ユーザー体験の流れ

1. ユーザー名 + パスワードだけでアカウント作成
2. テキストを書いて「投稿」を押す
3. AIが誤読による3単語要約を生成（バックグラウンド）
4. タイムラインには3単語だけが表示される
5. 気に入った投稿には推薦を送れる（1ユーザー最大3回）
6. 推薦はその日の中で集計される
7. 日次集計で最も推薦を集めた投稿が翌日に解放される
8. 原文は投稿者本人、または `Daily Recommend` に選ばれた投稿で閲覧できる

## 画面

| パス              | 内容                                            |
| ----------------- | ----------------------------------------------- |
| `/`               | タイムライン（3単語要約付き投稿一覧 + 投稿フォーム + 推薦案内） |
| `/recommend`      | 前日の日次集計で1位になった投稿の表示 |
| `/:noteId`        | 投稿の個別表示（本人、または `Daily Recommend` 対象のみ原文を閲覧可能） |
| `/login`          | ログイン                                        |
| `/register`       | アカウント登録（ユーザー名 + パスワードのみ）   |
| `/user/:username` | ユーザーの投稿一覧                              |

## 技術スタック

- **Frontend**: Vue 3 + Vite + Tailwind CSS
- **Backend**: Cloudflare Workers + D1 (SQLite)
- **API**: oRPC（Contract First で型安全）
- **AI**: Cloudflare Workers AI
- **認証**: ユーザー名 + パスワード → JWT

## 開発アプローチ

**Contract First** — API の型定義（契約）を最初に書き、サーバーとクライアントがそれに従う。
契約が Single Source of Truth。
